# Test Plan

## What
A structured outline of test coverage for a feature or system, identifying what to test, how, and at which layer.

## How
1. List acceptance criteria from the requirements
2. For each criterion: identify unit, integration, and E2E test candidates
3. Document data boundary cases: empty, null, max value, invalid format
4. Identify integration points that need contract testing
5. Flag performance and security test requirements
6. Estimate test coverage as a percentage of critical paths
