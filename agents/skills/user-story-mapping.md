# User Story Mapping

## What
User story mapping is a technique for organizing user stories into a two-dimensional map: the horizontal axis represents the user journey (activities in sequence), and the vertical axis represents priority (must-have at top, nice-to-have below). The map makes scope decisions visible and helps identify release slices.

## How
1. Start with the user narrative: write one sentence describing the end-to-end journey this map covers
2. Identify backbone activities (the major steps in the user journey, typically 5–10)
3. Under each activity, list the tasks users perform to complete it
4. Under each task, list user stories — small, testable increments of value
5. Write each user story in the format: "As a [persona], I want to [action] so that [outcome]"
6. Add acceptance criteria to each story: 2–4 concrete, testable conditions
7. Draw release slices horizontally — the top row of stories across all activities forms a releasable MVP
8. Annotate stories with: story points (if estimated), dependencies, open questions
9. Validate the map by walking through it as the user — does the sequence feel natural?
