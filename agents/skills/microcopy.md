# Microcopy

## What
The short text in UI that guides users through actions: labels, button text, error messages, empty states, tooltips, confirmation dialogs.

## How
1. Match the tone to the context (encouraging for onboarding, calm for errors)
2. Lead with the user action or outcome — not the system state
3. Error messages: what happened + what to do next (never just "Error")
4. Empty states: explain what goes here + provide a clear call to action
5. Button labels: verb + object ("Save changes", not "OK")
6. Keep it under 10 words where possible
